What's inside an .imf container

An IMF container is a sealed archive. You put files in, seal it, and the result is cryptographically locked — signed, hashed, and optionally timestamped on the Bitcoin blockchain.

Signature

Ed25519 digital signature ties the container to your identity. If someone else sealed it, the signature won't match your key.

Integrity

SHA-256 hash of every file. Change a single bit — in any file, in any metadata — and verification fails.

Timestamp

Bitcoin blockchain anchor via OpenTimestamps. Decentralized proof of when the container was sealed. No certificate authority, no annual fees.

Encryption

AES-256-GCM encryption (optional). For containers that need to stay private until you're ready to share.

How it works

# Create a new container
imf create photos-2026-02-23

# Add files
imf add photos-2026-02-23 IMG_001.jpg IMG_002.jpg scene.mp4

# Seal it — signs with your key, hashes everything
imf seal photos-2026-02-23

# Anchor to Bitcoin (optional, requires internet once)
imf timestamp photos-2026-02-23.imf

# Verify — works offline, no account needed
imf verify photos-2026-02-23.imf
# ✓ Signature valid (key: 7f3a...b2c1)
# ✓ All files intact (3 files, 0 modified)
# ✓ Bitcoin anchor confirmed (block 883,241)
Create & add files

Photos, documents, video, audio — any file type. From any camera or device.

Seal the container

Your Ed25519 key signs the container. SHA-256 hashes lock every file. The result is a single .imf file.

Anchor to Bitcoin

The container's hash is submitted to the Bitcoin blockchain via OpenTimestamps. Confirmation takes a few hours. After that, the timestamp is permanent.

Share and verify

Send the .imf file to anyone. They open it in IMF Viewer and see the signature, file integrity check, and blockchain timestamp. No account or internet needed for verification.

Who uses this

Journalism

Seal raw photos at the end of each day. If the images are later questioned, the blockchain timestamp proves they existed in that exact state on that date.

Legal Evidence

Photograph a scene, seal on a laptop before leaving. The cryptographic chain creates verifiable custody under Federal Rules of Evidence 902(13)–14.

Insurance

Document damage and seal immediately. The timestamp proves the documentation existed before any repair work began.

Creative Work

Seal work-in-progress at milestones. Each container is a timestamped proof of creation for IP disputes.

Whistleblowing

Encrypt sensitive documents in a sealed container. The blockchain anchor proves they existed before the investigation started.

Any file, any time

Contracts, research data, source code, medical records. If you need to prove a file existed unchanged at a point in time, IMF does it.

IMF vs. C2PA

C2PA is the industry standard for content provenance, backed by Adobe, Microsoft, Sony, and others. It's good work. But it covers a different part of the problem.

C2PA IMF
Signs at capture Yes, if hardware supports it No — seals post-capture
Works with Sony, Leica, Nikon (few models) Any camera or device
Timestamp TSA server (centralized) Bitcoin blockchain
Survives sharing No — metadata stripped by platforms Yes — the container is the file
Trust model X.509 certificates ($50–$500/yr) Ed25519 keys (free)
Verify offline Partial Full
File types Images, video, audio, docs Anything
Proves device origin Yes No — proves who sealed, and when

They're not competing standards. C2PA proves the camera. IMF proves custody and time. A photo with both has the strongest provenance story you can build.
